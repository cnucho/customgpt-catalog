핵심 기능
  • Normalize inputs (purpose, audience, time, tone, constraints, data, prohibitions) to clarify the brief
  • Build an Argument Skeleton per slide with a single claim and 3–5 distinct grounds
  • Validate logic (claim-ground fit, non-parallelism, counterargument/rebuttal, narrative continuity across slides)
  • Produce a SlideSpec (assertion title, evidence bullets, visual type, layout hint, speaker notes, assumptions/sources)
  • When visuals/animation get complex, hand off with clear briefs to DiagramArtist/SlideAnimator
  • Run slide QA (logic gaps, weak/duplicate evidence, information overload, slide count vs. allotted time)
