Functions

  • Phase A: 초기 탐색에서 후보 모형 2~3개를 균형 있게(장점/한계 포함) 제시
  • Phase B: 자료·목표·제약·활용처·윤리/품질 축을 따라 적합도 점수를 갱신하는 적응형 질의
  • Phase C: 단독/수정/결합 결정 후 Mermaid 1개 블록(한글 라벨)과 JSON 스키마 기반 최종 산출물 생성
  • 근거 우선순위(동료심사 논문→정부보고서→공식통계)에 따라 근거 제시 및 출처 품질 통제
  • 리뷰 모듈 절차(용어 정의→하위차원 누락→일관성→최신성→보완 제안)로 자체 점검 및 개선안 제시

Target users

  • 학술 연구자(이론/개념 모형 설계 필요)
  • 정책/공공 부문 분석가(설명가능한 프레임워크 필요)
  • 기업 전략/기획 담당자(문제구조화·프레임 설계 필요)
  • 컨설턴트/PM(요구사항 정리 및 모형화 필요)

Ideal use cases

  • 연구주제에 적합한 이론 모형(TAM/TPB/UTAUT 등) 후보를 신속히 비교·선정
  • 정책/전략 과제에서 SWOT/PESTEL/Logic Model 등 프레임워크 후보를 구조적으로 검토
  • 혼합방법(정량·정성) 설계에서 핵심 구성요소·하위차원 정의 및 누락 점검
  • 최종 산출물을 Mermaid 다이어그램과 구조화 JSON로 문서/보고서에 바로 반영

Limitations

  • Phase A에서는 단일 후보에 대한 심층 설명이나 조기 결론(선택 유도)이 제한됨
  • 분석 ‘방법론’(통계/ML)은 원칙적으로 후보 모형으로 다루지 않으며, 필요 시 검증 수단으로만 언급
  • 사용자 입력(자료 형태·규모·목표·제약·활용처·윤리/품질)이 부족하면 정교한 결정(Phase C)이 어려움
  • 근거 품질 기준에 따라 일부 출처(위키/개인블로그/비검증 매체)는 배제됨

Example commands

  • 내 연구는 설문(리커트) 기반이며 ‘행동의도’ 예측이 목표야. Phase A 후보 3개만 제시해줘.
  • 정책평가 과제인데 이해관계자 인터뷰(정성)도 있어. Phase B 질문으로 축 정보를 채워가자.
  • 지금까지 정보로 Phase C에 들어가서 ‘수정’ 또는 ‘결합’ 결정과 Mermaid+JSON 산출물을 만들어줘.
  • 내 모형 구성요소 정의가 겹치는지 리뷰 모듈 기준으로 점검하고 보완 제안을 목록으로 줘.

Additional features

  • 후보 모형 근거 우선순위 및 금지 출처 정책 내장
  • 최종 산출물 JSON 스키마(components/justification/mermaid_code/review_suggestions) 준수
  • 자체 리뷰 모듈 기반의 개선 제안 자동 생성
