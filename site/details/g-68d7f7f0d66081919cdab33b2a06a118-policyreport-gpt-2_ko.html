Functions

  • 5단계 고정 흐름(problem_definition→comparison→alternatives→execution_plan→conclusion)으로 정책 보고서 초안을 구조화
  • 각 단계별: 1단락 요약 + 1개 표/매트릭스 산출
  • 단계 종료 시 report_state를 YAML 블록으로 갱신·출력(현재 단계 결과만 반영)
  • 정량 정보 표기 시 ‘수치+단위+출처’ 형식을 강제하여 근거 중심 작성 지원
  • 사용자 범위/입력에 맞춰 비강압적 진행(과도한 제안·추가 과제 유도 최소화)

Target users

  • 정책기획·전략·공공/민간 싱크탱크 연구자
  • 정부·지자체·공공기관 실무자
  • 기업 정책/대관·컴플라이언스 담당자
  • 컨설턴트·PM·사업기획자

Ideal use cases

  • 정책/전략 과제의 문제정의 및 범위 고정(목표, 제약, 이해관계자 정리)
  • 대안 비교표(효과, 비용, 리스크, 실행가능성 등) 작성
  • 실행계획(로드맵, 일정, 책임, 리스크 완화) 초안 생성
  • 보고서 문서화를 위한 일관된 단계별 산출물 및 YAML 상태관리

Limitations

  • 사용자가 제공하지 않은 사실·수치·출처는 임의로 확정하지 않으며, 최신 정보가 필요한 경우 별도 확인이 필요할 수 있음
  • 조직 내부 규정/법률자문이 필요한 고위험 의사결정은 최종 책임 있는 검토가 필요
  • 템플릿/단계 구조를 우선하므로 매우 자유형 서술 문체에는 덜 적합할 수 있음

Example commands

  • 다음 정책 이슈를 5단계 프레임으로 정리해줘: (배경/목표/제약/대상/기간)
  • 대안 3개를 효과·비용·리스크·실행가능성 기준으로 비교표로 만들어줘
  • 선정된 대안에 대해 12개월 실행계획(분기별 마일스톤, 담당, 예산 가정)을 작성해줘
  • 각 단계 끝에 report_state YAML도 함께 출력해줘(현재 단계 결과만 반영)

Additional features

  • 단계별 산출물 형식 고정(요약 1단락 + 표 1개)으로 보고서 일관성 강화
  • 비강압적 상호작용 원칙(사용자 범위 이탈 최소화, 과도한 다음 단계 유도 지양)
  • 상태(YAML) 기반 진행 관리로 협업/재현/감사 추적에 유리
